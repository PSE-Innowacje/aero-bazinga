### Task 2: Create OperationsListPage
type: auto

Create client/src/pages/operations/OperationsListPage.tsx:

### Task 3: Create OperationFormPage (create + edit)
type: auto

Create client/src/pages/operations/OperationFormPage.tsx:

Used for both creating (no :id param) and editing (/operations/:id/edit)

Fields:
- project_reference (text, max 30, required)
- short_description (text, max 100, required)
- operation_type_ids (multi-select checkboxes, min 1 required, loaded from GET /api/operations/types/list)
- proposed_earliest_date (date, optional)
- proposed_latest_date (date, optional)
- additional_info (textarea, max 500, optional)
- contact_emails (comma-separated text input or list of email inputs, optional)
- kml_file (file input, accept=".kml", required on create, optional on edit)

Supervisor-only fields (hidden for planner):
- planned_earliest_date (date)
- planned_latest_date (date)
- post_completion_notes (textarea, max 500) — shown only when status >= 5

Auto-calculated (read-only, shown in edit mode):
- operation_number (read-only, display)
- route_distance_km (read-only, display in km)
- status (read-only badge, not editable via form — use status action buttons)
- created_by_email (read-only, display)

Submit uses multipart/form-data (FormData) for KML upload.

After save: navigate to /operations/:id (detail view)

### Task 4: Create OperationDetailPage (view with map, history, comments, status actions)
type: auto

Create client/src/pages/operations/OperationDetailPage.tsx:

Layout sections:
1. Header: operation_number, status badge, action buttons
2. Operation details (read-only fields in a grid)
3. Interactive Leaflet map showing KML route points as polyline
4. Linked flight orders list (OPS-06c, read-only)
5. Comments section (list + add comment form)
6. History log (expandable/collapsible table)

Action buttons (per role and status):
- Supervisor, status=1: "Odrzuć" button (1→2) + "Potwierdź" button (1→3)
- Planner or Supervisor, status ∈ {1,3,4}: "Anuluj operację" button (→7)
- "Edytuj" button: shown for planner (status 1,2,3,4,5) and supervisor (all statuses)

For "Potwierdź": if planned dates not set, show warning that they need to be set first (supervisor can edit them then confirm).

Map requirements:
- Import leaflet CSS: import "leaflet/dist/leaflet.css"
- Fix Leaflet default icon issue (delete L.Icon.Default.prototype._getIconUrl, set iconUrl/shadowUrl manually)
- Center map on points centroid or Poland center (52.0, 19.0) if no points
- Draw polyline in primary blue (#003E7E)
- Add small circle markers at each point
- Responsive height (min 300px)

Wrap Leaflet map in a try/catch or lazy mount (SSR safe) using useEffect to initialize.
